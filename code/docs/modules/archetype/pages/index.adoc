<h1>Archetype Karate E2E</h1>
<p>Arquetipo Maven para generar:</p>
<ul>
<li>Módulo siempre: <code>e2e/karate</code> con tests Karate.</li>
<li>Módulo opcional: <code>code</code> (micro Spring Boot) controlado por <code>includeCodeModule</code>.</li>
</ul>
<h2>1. Instalar el arquetipo localmente</h2>
<p>Desde la raíz del repo que contiene <code>code/archetype-karate-e2e</code>:</p>
<pre><code class="language-bash">mvn -f code/archetype-karate-e2e clean install
</code></pre>
<h2>2. Generar proyecto SIN micro (modo offline por defecto)</h2>
<pre><code class="language-bash">mvn archetype:generate \
  -DarchetypeGroupId=com.example.archetypes \
  -DarchetypeArtifactId=archetype-karate-e2e \
  -DarchetypeVersion=1.0.0-SNAPSHOT \
  -DgroupId=com.acme \
  -DartifactId=demo-e2e-no-micro \
  -Dversion=1.0.0-SNAPSHOT \
  -Dpackage=com.acme.demo \
  -DincludeCodeModule=false \
  -B

cd demo-e2e-no-micro
mvn -pl e2e/karate test
</code></pre>
<p>El proyecto generado incluye un feature OFFLINE (sin llamadas HTTP) para garantizar que los tests pasan incluso en entornos restringidos.</p>
<p>Ejecutar:</p>
<pre><code class="language-bash">mvn -pl e2e/karate test
</code></pre>
<h3>(Opcional) Activar mock embebido</h3>
<p>Dentro del feature <code>getProductsLocal.feature</code> hay un escenario comentado <code>(ejemplo mock embebido)</code>. Descoméntalo si tu entorno soporta <code>karate.start()</code>:</p>
<pre><code class="language-gherkin">Scenario: (ejemplo mock embebido)

<p>Si obtienes <code>Connection refused</code>, deja el escenario comentado y permanece en modo offline.</p>
<h2>3. Generar proyecto CON micro</h2>
<pre><code class="language-bash">mvn archetype:generate \
  -DarchetypeGroupId=com.example.archetypes \
  -DarchetypeArtifactId=archetype-karate-e2e \
  -DarchetypeVersion=1.0.0-SNAPSHOT \
  -DgroupId=com.acme \
  -DartifactId=demo-e2e-with-micro \
  -Dversion=1.0.0-SNAPSHOT \
  -Dpackage=com.acme.demo \
  -DincludeCodeModule=true \
  -B
</code></pre>
<p>Levantar micro (puerto 9090 por ejemplo):</p>
<pre><code class="language-bash">cd demo-e2e-with-micro
export APP_PORT=9090
mvn -pl code -am spring-boot:run -Dspring-boot.run.arguments=--server.port=$APP_PORT
</code></pre>
<p>En otra terminal:</p>
<pre><code class="language-bash">cd demo-e2e-with-micro
export APP_PORT=9090
mvn -pl e2e/karate test
</code></pre>
<p>En Windows PowerShell:</p>
<pre><code class="language-powershell">$env:APP_PORT=9090
mvn -pl e2e/karate test
</code></pre>
