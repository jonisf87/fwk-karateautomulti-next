<h1>Karate E2E</h1>
<p>Suite E2E del repositorio, basada en Karate 2.0.2 y Java 21.</p>
<h2>Overview</h2>
<p>La suite separa dos modos de ejecucion:</p>
<ul>
<li><code>~@local</code>: validacion estandar del repositorio, sin depender del microservicio local</li>
<li><code>@local</code>: flujo real del desarrollador contra el servicio demo levantado manualmente</li>
</ul>
<h2>Requirements</h2>
<ul>
<li>Java 21</li>
<li>Maven 3.9 o superior</li>
</ul>
<h2>Installation</h2>
<pre><code class="language-bash">cd e2e/karate
mvn clean verify &#39;-Dkarate.options=--tags ~@local&#39;
</code></pre>
<h2>Execution</h2>
<p>Flujo remoto estandar:</p>
<pre><code class="language-bash">mvn clean verify &#39;-Dkarate.options=--tags ~@local&#39;
</code></pre>
<p>Flujo local:</p>
<pre><code class="language-bash">mvn clean verify -Dkarate.env=local &#39;-Dkarate.options=--tags @local&#39;
</code></pre>
<p>Flujo local con puerto personalizado:</p>
<pre><code class="language-bash">APP_PORT=18081 mvn clean verify -Dkarate.env=local &#39;-Dkarate.options=--tags @local&#39;
</code></pre>
<p>Filtrado por tags:</p>
